# Why TDD? 
Test-Drien Development

**Red** : Write failing test   
**Green** : Make it pass   
**Refactor** : Improve the code   

Given/When/Then

(+) 소스코드 품질 향상. 필요한 코드만 작성하고 테스트르 통과한 코드만 존재하믈 디버깅 시간 절약  
(+) 모듈화 디자인, 유지보수 용이, 리팩토링 용이, 테스트 코드 문서화   

(-) 프로토타이핑으 바른 시간 안에 마치고 결과를 보여줘야하는 상황에서 부적합  
(-) 적응
